КОНТРОЛЬНЫЕ ВОПРОСЫ

1. Структура типового пакета

Типовой пакет содержит следующие поля:

ПолеНазначение
ПреамбулаНастройка аппаратуры на прием, синхронизация

Адрес отправителяИдентификатор абонента-отправителя

Поле данных может отсутствовать в управляющих пакетах. Преамбула и стоповая комбинация могут отсутствовать при использовании самосинхронизирующихся кодов.

2. Алгоритм протокола обмена информацией

Базовый протокол обмена (сеанс связи):

  1. Передатчик → Запрос готовности → Приемник
  2. Приемник проверяет готовность:
    • • Если не готов → Отказ (конец сеанса)
    • • Если готов → Готовность → Передатчик
  3. Цикл передачи данных:
    • • Передатчик → Пакет данных → Приемник
    • • Приемник проверяет контрольную сумму
    • • Если ошибок нет → Подтверждение
    • • Если ошибка → Запрос повторной передачи
  4. Передатчик → Конец связи → Приемник

Используются управляющие пакеты: Запрос, Готовность, Подтверждение, Конец.

3. Алгоритм расчета оптимальной длины пакета

Шаг 1. Сбор исходных данных: l, C, Sн, K₁, tн, p_бит.

Шаг 2. Расчет ω₂ с точки зрения памяти:

ω₂ = √(l · C); ω₂' = K₁ · ω₂

Шаг 3. Для каждого ω из заданного ряда (512,1024,2048,4096,8192,16384):

p_пакета = 1 - (1 - p_бит)ω

Sэ = (ω - C) / (ω/Sн + tн) · (1 - p_пакета)

Выбирается ω₃ с максимальной Sэ.

Шаг 4. Определение оптимальной длины:

ω* = (ω₂' + ω₃) / 2

m = round(log₂(ω*)); ω*_опт = 2m

Полученное значение обеспечивает компромисс между долей служебной информации и вероятностью ошибки.
